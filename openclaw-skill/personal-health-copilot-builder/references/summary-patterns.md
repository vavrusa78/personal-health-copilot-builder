# Summary patterns

## Summary outputs should be derived

Never compute totals only inside prose. Summaries should read from deterministic derived data.

## Useful outputs

### Running daily status
Use after new meals, drinks, or measurements.

Should answer:
- where the user is versus plan
- what is missing
- what remains uncertain

### Morning recap
Use for sleep, recovery, and overnight biometrics.

Should be:
- short
- trend-aware
- non-diagnostic

### Evening recap
Use for completed day summary.

Should cover:
- intake totals
- hydration
- activity if tracked
- notable signals

### Weekly trend
Use only after enough data exists.

Should focus on:
- repeated patterns
- changes versus baseline
- uncertainty when sample size is small

## Presentation variants

Safe order:
1. text recap
2. markdown report
3. cards
4. video

Video is presentation, not core logic.
